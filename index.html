Git Bash

If you're on a Windows device and will be using the terminal within your editor, it may be a good idea to also install Git Bash. Windows users can perform command-line operations using the Command Prompt or Powershell, and some developers swear by them, but Git Bash works much more like what you would see on a Mac or Linux environment. It is specifically designed to work with git and is easy to use.

What IS Git?

Git is a Distributed Version Control System. It is the most widely used DVCS in the world. Git is NOT GitHub, but GitHub works off of Git. More on that later.

Version Control

Version control is all about keeping track of changes to software. Before version control, you often had to save several different previous copies of the same file if you wanted to revert to a previous change. And what if you wanted to revert several changes all at once? And if you're working on a team, how do you know who changed what and when? With Git, you only have one file you're working on, and every time you "commit" a change, it stores the previous version of the file you altered as well as a lot of meta data about the changes. You will know who changed what, when, and, if they're doing it right, why.

Distributed

Distributed means that there is a full copy of the repository on every computer that has cloned it. This copy even includes ALL of the history. Every change made by every team member can be accessed on your local computer, even without an internet connection. Changes made to the online repository will still need to be downloaded to your local computer, but there is a process for that. You can still make your changes offline on your local device and then push to your teams online repository when you're online.

Using Git

To Terminal, or Not To Terminal

When working with Git in VS Code you have the option of using either a terminal (or command line) interface or by using Codes integrated git controls. If you're a little intimidated by using the command line at first, that's fine. However, you will probably be expected to know the terminal commands by a future employer so it is recommended you start practicing with it soon. Both methods will be covered side-by-side in this guide so if you want to work with the integrated controls at first to get a fundamental understanding of what you're doing, you can then go back and level up later by comparing the methods side-by-side.
